# Product And UX

## 1. Product Idea

I built Learn Kyrgyz as a mobile-first language learning app for short daily practice. I want the experience to stay focused, calm, and repeatable instead of feeling like a collection of disconnected screens.

The current learning experience is based on:

- flashcards
- quizzes
- sentence building
- category-based progression
- streak and milestone motivation
- local progress with cloud sync support

## 2. Main User Scenarios

### New user

1. I open the app.
2. I go through the welcome flow.
3. I choose a daily goal.
4. I continue as a guest or sign in.
5. I land on Home and see one clear next step.

### Returning user

1. I open the app again.
2. I use Home as the “what should I do now?” screen.
3. I continue my last learning task or close my review queue.
4. I finish a short study cycle and move to the next suggested action.

## 3. Screen Roles

### Home

I treat Home as a decision screen with one purpose: show the best next action.

Home is responsible for:

- continue last lesson
- review due entry
- daily goal visibility
- milestone awareness

### Practice

I use Practice as a tactical hub, not as a second home page.

Practice is where I choose a mode intentionally:

- flashcards
- quick quiz
- sentence builder
- review-focused entry points

### Categories

I use Categories as the category roadmap.

This screen should show:

- where the learner is now
- what is already completed
- what is unlocked next
- where review pressure exists

### Flashcards, Quiz, and Sentence Builder

I treat these as connected parts of one learning loop.

The intended sequence is:

1. recall
2. check
3. apply
4. review mistakes

### Progress and Profile

I do not want fake analytics here.

These screens are meant to show:

- streak
- mastered words
- weak words
- review due
- next milestone
- sync status
- account and study settings

## 4. UX Rules I Follow

I use these rules across the project:

1. one primary CTA per screen
2. critical actions should stay visible without awkward scrolling
3. Home and Practice must have clearly different roles
4. study direction belongs to settings, not to every exercise screen
5. review due should be part of the main experience
6. progress should depend on real tracked data

## 5. Current Learning Loop

The current product direction is:

1. choose a category or continue the last one
2. study with flashcards or review mode
3. move into quiz
4. reinforce through sentence builder
5. get a summary and a next step
6. send weak words back into review

## 6. What I Already Improved

- I separated the roles of Home, Practice, and Categories more clearly.
- I added review mode for flashcards.
- I improved milestone and progress surfaces.
- I added local analytics for study sessions.
- I improved adaptive mobile layouts and CTA visibility.

## 7. What Still Needs Work

### Priority 1

- a dedicated review hub
- stronger mistake feedback in quiz and sentence builder
- clearer category progression

### Priority 2

- weekly recap based on real history
- full interface localization
- stronger tablet and web layouts

### Priority 3

- richer pronunciation support
- better achievement claiming flow
- more personalized Home behavior based on review debt and momentum

## 8. UX Constraints

I do not want to:

- add decorative CTAs with no real action
- rely on fake analytics
- turn Home into a catalog of features
- overload study screens with settings and extra chrome
- document future ideas as if they already exist
