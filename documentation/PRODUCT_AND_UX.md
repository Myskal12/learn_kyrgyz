# Product And UX

Обновлено: 2026-04-01

## 1. Что это за продукт

Learn Kyrgyz - mobile-first приложение для ежедневной практики кыргызского языка через:

- flashcards
- quiz
- sentence builder
- category-based learning path
- streak и milestone-мотивацию
- progress sync между guest/local и cloud account

Главная цель продукта - довести пользователя до короткой, понятной и повторяемой ежедневной практики, а не просто показать набор красивых экранов.

## 2. Основные продуктовые сценарии

### Новый пользователь

1. Открывает приложение.
2. Проходит welcome flow.
3. Выбирает дневную цель.
4. Продолжает как guest или входит в аккаунт.
5. Попадает на Home и получает один лучший следующий шаг.

### Возвращающийся пользователь

1. Открывает приложение.
2. Видит Home как экран "что делать сейчас".
3. Либо продолжает прошлую сессию, либо закрывает review due.
4. После упражнения получает summary и следующий рекомендуемый шаг.

## 3. Роли основных экранов

### Home

Home не должен быть вторым Practice и не должен быть витриной всего приложения.

Он отвечает только на вопрос:
"Что мне сделать прямо сейчас?"

На Home приоритетны:

- continue last lesson
- review due
- daily goal progress
- next milestone

### Practice

Practice - это tactical hub, а не еще одна главная страница.

Он нужен для осознанного выбора режима:

- flashcards
- quick quiz
- sentence builder
- review-focused entry points

### Categories

Categories - это roadmap обучения по категориям.

Экран должен объяснять:

- где пользователь сейчас
- что уже завершено
- что доступно следующим
- где требуется повторение

### Flashcards / Quiz / Sentence Builder

Это три разных режима одной учебной системы, а не три изолированных мини-приложения.

Ожидаемая последовательность:

1. изучить / вспомнить
2. проверить
3. применить в контексте
4. вернуться к ошибкам

### Progress / Profile

Эти экраны не должны показывать фейковую аналитику.

Они нужны для:

- streak
- mastered / weak / review due
- next milestone
- sync state
- базовых пользовательских настроек

## 4. Текущий UX-курс проекта

В проекте уже приняты и частично реализованы следующие правила:

1. Один главный CTA на экран.
2. Критические действия не должны прятаться в длинный scroll.
3. Home и Practice должны быть четко разведены по роли.
4. Настройка направления обучения живет в settings, а не перегружает учебные экраны.
5. Review due должен быть частью основного опыта, а не побочной функцией.
6. Progress должен быть честным и опираться на реальные данные.

## 5. Текущий учебный цикл

Сейчас продукт двигается к следующей модели:

1. Пользователь выбирает категорию или продолжает последнюю.
2. Проходит flashcards или review mode.
3. Переходит в quiz.
4. Закрепляет конструкцию через sentence builder.
5. Получает summary и следующий шаг.
6. Проблемные слова уходят в review loop.

Это уже лучше прежней разрозненной модели, но цикл еще не завершен полностью.

## 7. Что еще остается улучшить

### P1

- отдельный review hub вместо распределенных review entry points
- более сильная обратная связь по ошибкам в quiz и sentence builder
- более прозрачная прогрессия категорий

### P2

- weekly recap на базе реальной истории, а не заглушек
- полноценная локализация интерфейса
- tablet/web layout beyond mobile scaling

### P3

- richer audio / pronunciation experience
- stronger achievement claiming flow
- более умная персонализация home по momentum и review debt

## 8. UX-ограничения, которые важно помнить

- Нельзя добавлять декоративные CTA без реального действия.
- Нельзя строить мотивацию на fake analytics.
- Нельзя превращать Home в каталог всех возможностей.
- Нельзя перегружать учебные экраны настройками и лишним chrome.
- Нельзя писать документацию "про будущее", если этого еще нет в коде.
