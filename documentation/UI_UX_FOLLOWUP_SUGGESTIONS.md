# UI/UX Follow-Up Suggestions

## Learn Kyrgyz

Этот список дополняет уже внедрённые исправления и нужен как следующий backlog после базовой UX-стабилизации.

## P1: Следующие самые полезные улучшения

1. Сделать прогрессию категорий самодокументируемой.
   На карточке категории явно показывать:
   - сколько слов уже освоено
   - сколько слов осталось
   - почему следующая категория закрыта
   - что откроется после завершения текущей

2. Добавить полноценный review queue.
   Сейчас приложение уже показывает слова, которые требуют повторения, но это не отдельная система. Нужен единый экран или блок "На повторении", который агрегирует:
   - слова с низкой точностью
   - незакрытые ошибки из квиза
   - проблемные предложения

3. Упростить auth-формы до product-grade UX.
   Полезно добавить:
   - inline validation у email и password
   - понятные ошибки под полями
   - требования к паролю до submit
   - autofill и более явный guest CTA

4. Добавить восстановление серии.
   Если пользователь пропустил день, можно показывать:
   - "серия потеряна"
   - "вернитесь сегодня, чтобы начать заново"
   - мягкий recovery CTA вместо сухого обнуления

5. Улучшить onboarding микро-копирайтинг.
   Стоит сделать более action-first copy для:
   - welcome
   - home hero
   - practice scenarios
   - summary экранов

## P2: Визуальная и системная полировка

6. Пересобрать адаптивность для tablet и web.
   Следующий шаг:
   - расширенный shell после `>= 768px`
   - двухколоночные layouts в progress/practice/profile
   - более широкие summary cards

7. Вынести единый компонент progress header.
   Сейчас progress bars и header-паттерны уже лучше, но всё ещё распределены по разным экранам. Нужен единый reusable block для:
   - текущего шага
   - процента
   - supporting hint

8. Усилить accessibility.
   В первую очередь:
   - `Semantics` для action buttons и progress
   - screen-reader labels на quiz answers
   - text scaling smoke test
   - touch target audit

9. Добавить haptic/audio feedback strategy.
   Особенно полезно для learning flow:
   - мягкий success haptic
   - distinct error haptic
   - единая политика voice/audio CTA

10. Нормализовать visual hierarchy.
   Дальше стоит унифицировать:
   - размеры заголовков по уровням
   - насыщенность градиентов
   - вторичные кнопки и text links
   - плотность карточек в длинных списках

## P3: Более сильные продуктовые идеи

11. Ввести mastery bands вместо одного процента.
   Например:
   - new
   - seen
   - practicing
   - mastered
   Это сделает прогресс более понятным и учебно-ориентированным.

12. Добавить lesson preview перед стартом.
   Для каждой категории можно показывать:
   - сколько слов внутри
   - какой тип практики доступен
   - сколько времени примерно займёт проход

13. Сделать achievements более “claimable”.
   Вместо просто списка:
   - новый unlock banner
   - badge detail sheet
   - переход из summary в achievement state

14. Добавить weekly recap.
   Если позже появится реальная история по дням, можно сделать:
   - weekly recap card
   - best category of the week
   - words that still need review

15. Поддержать social-lite мотивацию.
   Не обязательно полноценная социальная сеть. Достаточно:
   - сравнение с собой за прошлую неделю
   - “вы выросли на X слов”
   - leaderboard context around user position
