# Wireframes and Prototypes

**Project:** Learn Kyrgyz

---

## UX Objectives

- Minimize time to first lesson
- Simple daily practice for returning users
- Core actions in 1–2 taps
- Clear offline states
- Motivation via progress and achievements

---

## Design Principles

1. Clarity first — one primary action per screen
2. Progressive disclosure — advanced actions when needed
3. Fast feedback — taps, scoring, sync status
4. Accessibility — readable typography, high contrast, 44px touch targets
5. Cultural relevance — Kyrgyz-centered examples

---

## Information Architecture

**Primary:** Home, Categories, Practice, Achievements, Profile  
**Secondary:** Onboarding, Leaderboard, Settings, Study plan

---

## Screen Inventory

**Onboarding:** Splash, welcome carousel, sign-in/register, Google, guest, password reset.

**Home:** Dashboard, continue learning, category highlights, streak panel, quick actions.

**Learning:** Category list, word list, flashcard session, quiz screen, results, sentence practice.

**Progress:** Progress dashboard, achievements, leaderboard, rank details.

**Profile:** Overview, edit nickname/avatar, theme/language, notifications, account.

---

## Key Prototype Flows

**Flow A:** New user → welcome → sign-in → home → category → first flashcard.  
**Flow B:** Quick quiz → 5 questions → score and review → save progress.  
**Flow C:** Achievements → badge details → leaderboard → home.

---

## Deliverables

Low-fidelity wireframes, mid/high-fidelity key flows, clickable prototype, UX annotations, usability test script.
